# web-gateway

`web-gateway` 是一个面向 AI Agent 的联网路由 skill。它不是把所有网页任务都交给单一爬取方式，而是把 OpenCLI、Agent 自带检索/网页读取能力，以及浏览器自动化后端统一到同一套决策流程中。

现在常见的 AI 联网工具各有侧重：

| 工具 | 适合场景 | 局限 |
|------|----------|------|
| OpenCLI | 对知乎、小红书等已适配站点，在浏览器登录态下进行结构化读取、评论、发帖等操作 | 依赖站点 adapter，未适配站点无法覆盖 |
| Web Access / 浏览器自动化 | 在用户浏览器登录态下读取动态网页、执行 DOM 操作、截图、填表、上传文件 | 原始页面信息可能混杂；对特定站点语义和业务动作缺少定制化 |
| Agent 自带检索 / 网页读取 | 公开网页搜索、正文提取、资料核实，通常结果更干净 | 不携带用户登录态，不能操作页面，也不能处理需要交互的内容 |

`web-gateway` 的核心目标是把这些能力组合起来：OpenCLI 能覆盖的特定站点优先使用 OpenCLI；公开网页、无需登录态和交互时优先使用 Agent 自带检索/网页读取；需要登录态、动态页面、DOM 读取、表单填写、文件上传、视频帧采样等能力时，再进入浏览器后端。

本 fork 在 Web Access 的浏览器能力基础上，增加了浏览器扩展后端和本地路由策略。扩展后端一次安装授权后，通过本地 `127.0.0.1` daemon 与 Chrome / Edge 通信，减少反复触发 remote-debugging 授权弹窗的成本；CDP proxy 仍作为兜底方案保留。
